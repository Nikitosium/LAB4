# RS-31_Haponko_Nikita_RTOS_LR4

**Номер групи:** РС-31

**Автор:** Гапонько Нікіта

**ОПЕРАЦІЙНІ СИСТЕМИ РЕАЛЬНОГО ЧАСУ ТА 32-БІТНІ МК**
**Лабораторна робота 4.** Технологія використання апаратних таймерів та переривань при розробці ПЗ ОСРЧ.

**Обладнання:** STM32F411 Discovery, CubeMX, Keil.

**Мета:** Вивчити принципи роботи апаратних таймерів мікроконтролера STM32, навчитися налаштовувати переривання та генерувати сигнали з точними часовими характеристиками без використання блокуючих затримок.

__________________________________________________________________________________________

**Завдання до лабораторної роботи (Варіант 4):**
1. Реалізувати формування сигналу на **PD14** з частотою **100 Гц** та тривалістю імпульсу **5 мс**, використовуючи таймер **TIM2**.
2. Реалізувати формування сигналу на **PD15** з частотою **200 Гц** та тривалістю імпульсу **2 мс**, використовуючи таймер **TIM3**.

Цей проєкт демонструє перехід від програмних затримок до апаратних ресурсів МК. Використання переривань дозволяє основному циклу програми залишатися вільним для виконання інших завдань.

### 👨🏼‍💻 Функціонал:
* **Hardware Timing**: Використання подій оновлення таймерів для точного відліку мікросекундних інтервалів.
* **Interrupt-Driven Control**: Керування станом світлодіодів відбувається всередині обробників переривань (ISR).
* **Asynchronous Operation**: Обидва сигнали генеруються незалежно один від одного з різними частотами та шпаруватістю.

### 📋 Ключові моменти реалізації:
1. **Timer Configuration**: Розрахунок значень Prescaler та Counter Period для досягнення частот 100 Гц та 200 Гц.
2. **Interrupt Handlers**: Використання функцій зворотного виклику (Callbacks) для перемикання станів пінів PD14 та PD15.
3. **Efficiency**: Мінімальне навантаження на процесор, оскільки робота виконується лише в моменти спрацювання таймера.
